# Project 03: Smart Text Summariser

> **Bootcamp Day 3** · Tier 1: Foundation · Est. build time: 60 min

Paste any text and get an instant summary in your chosen format: a one-liner, bullet points, a full paragraph, or all three at once — powered by a local LLM via Ollama.

---

## What You Will Build

A single-page summariser where:

- The user pastes any text (article, meeting notes, documentation)
- Selects a format: One-Liner, Key Points, Full Summary, or All Formats
- Clicks Summarise — HTMX POSTs the form and injects the HTML response directly into the page
- Go renders a markdown summary with `goldmark`, returns an HTML fragment
- HTMX swaps the fragment into the result area with no JavaScript needed

---

## Key Concepts Introduced

| Concept | What It Teaches |
|---|---|
| HTMX form submission | `hx-post`, `hx-target`, `hx-swap`, `hx-indicator` — progressive enhancement without custom JS |
| HTML fragments from Go | Server returns `text/html` instead of `application/json`; HTMX injects it directly |
| Named Go templates | `{{define "index"}}` and `{{define "result"}}` in one file; `ExecuteTemplate` selects the right one |
| `goldmark` markdown rendering | Converting AI markdown output to HTML server-side (first external Go dependency) |
| Multi-format prompt design | One prompt, three output sections; robust section parsing with fallback |

## Architecture

```
Browser                      Go Server                    Ollama
  |                              |                            |
  | HTMX POST /summarize         |                            |
  | (text, format)               |                            |
  |----------------------------> |                            |
  |                              | ai.Chat(DefaultModel, msgs)|
  |                              | stream: false              |
  |                              |--------------------------> |
  |                              |                            |
  |                              | Full markdown response     |
  |                              | <------------------------- |

  |  </div>                      |                            |
  | <--------------------------- |                            |
  |                              |                            |
  | HTMX swaps into #result      |                            |
```

## Key Files Explained

### `handlers/summarize.go`

**The HTMX pattern** — the central difference from Projects 01 and 02:

```go
// Project 02: returned JSON
w.Header().Set("Content-Type", "application/json")
json.NewEncoder(w).Encode(explainResponse{...})

// Project 03: returns an HTML fragment
w.Header().Set("Content-Type", "text/html; charset=utf-8")
tmpl.ExecuteTemplate(w, "result", data)
```

HTMX receives the HTML string and swaps it into `<div id="result">`. No JavaScript parsing needed.

**Multi-format section parsing** — the `parseSections()` function splits the AI response by known headings (`## ONE-LINER`, `## KEY POINTS`, `## FULL SUMMARY`). It uses case-insensitive partial matching to tolerate minor variations in the AI's output, with a fallback that treats the full response as a summary if no headings are found.

**goldmark conversion** — `mdToHTML()` converts a markdown string to a `template.HTML` value. The `template.HTML` type bypasses Go's auto-escaping because the content is from a trusted local source (Ollama), not from user input.

### `templates/index.html`

Two named templates in one file:

```go
// Full page — served on GET /
tmpl.ExecuteTemplate(w, "index", nil)

// HTML fragment — returned to HTMX on POST /summarize
tmpl.ExecuteTemplate(w, "result", data)
```

The HTMX form attributes:

```html
<form
  hx-post="/summarize"    <!-- POST to this URL on submit -->
  hx-target="#result"     <!-- inject response into this element -->
  hx-swap="innerHTML"     <!-- replace inner HTML of the target -->
  hx-indicator="#spinner" <!-- show #spinner while request is in flight -->
>
```

## Why `goldmark` Instead of a JavaScript Renderer?

Projects 01 and 02 used marked.js on the frontend to convert markdown to HTML.
Project 03 uses `goldmark` in Go to do this server-side.

The reason is the HTMX pattern: we return an HTML fragment, not JSON.
If we returned a raw markdown string inside HTML, HTMX would render it as
plain text — the markdown syntax would appear literally on screen.
Go must convert the markdown to HTML before it leaves the server.

| | Project 02 | Project 03 |
|---|---|---|
| Response type | JSON | HTML fragment |
| Markdown rendering | marked.js (browser) | goldmark (Go server) |
| HTMX involvement | None | `hx-post`, `hx-target`, `hx-swap` |
